# Работа сервера:
Представляет собой приложение на Python, которое выполняет роль "мозга" робота. 

Его основная задача - объединить множество функций: 
- управление аппаратной частью (моторы, сервоприводы, датчики),
- обработку мультимедиа (видео, музыка, радио), 
- взаимодействие с пользователем (голос, жесты, лицо)
- обеспечение удаленного управления через сокетное соединение (с Android-приложения / IOS-приложения ).

# Многопоточность сервера

*Основной поток (GUI):* main.py запускает интерфейс PyQt5, который отображает видео с камеры, анимацию лица робота и виджеты плееров.

*Поток сервера:* Внутри main.py запускается объект Server, который создает свои потоки для работы с сетью.

*Поток видеозахвата:* Server запускает поток translate_video_on_screen, который постоянно читает кадры с камеры.

*Поток видеотрансляции:* Server запускает поток transmission_video, который ожидает подключения клиента и отправляет ему сжатые кадры.

*Поток команд:* Server запускает поток receive_instruction, который ожидает команд от клиента по отдельному сокету и выполняет их.

*Поток голосового ассистента:* Server запускает поток takeCommand, который в фоне слушает микрофон, активируется по ключевому слову и выполняет голосовые команды.

# Описание файлов

## 3. *main.py*

Исполняемый файл. GUI 
- Создает окно Ui_MainWindow (файл server_interface.py)
- Управляет отображением видеопотока/ конвертирует его
- Управляет анимацией "лица" (GIF). Переключает анимации в зависимости от состояния робота
- Запускает серверные потоки и голосового ассистента.
- Отслеживает события мыши/клавиатуры, чтобы сбрасывать таймеры ожидания

## 4. *Server.py*

Координирует работу всех модулей и обеспечивает связь с клиентом.
- Создает экземпляры всех сервисов (Service), модуля UART, анализатора ключевых фраз и т.д.
- turn_on_server(): Запускает два сокета: для видео (порт 8001) и для команд (порт 5001).
- transmission_video(): Работает в отдельном потоке. Принимает подключение от клиента, сжимает текущий кадр в JPEG и отправляет его.
- receive_instruction(): Работает в отдельном потоке. Принимает команды от клиента. Парсит их и вызывает соответствующие методы (управление движением, мультимедиа, калибровка и т.д.), используя объекты Service, UART и ServoControl. Отправляет обратно клиенту подтверждения или данные.
- takeCommand(): Работает в отдельном потоке. В бесконечном цикле слушает микрофон через service.recognizer. При обнаружении ключевого слова "избушка" активируется и передает распознанную фразу в метод recognizing() для выполнения.

## 5. *Service.py*

При инициализации создает объекты всех остальных сервисных модулей: Speaker, FaceAction, HandAndFingerDetectionModule, MusicPlayer, VideoPlayer, RadioPlayer, Recognizer, DatabaseManager, PlayVideoViaBrowser, KeyPhraseAnalizer, GeminiAssistant.

Содержит методы, которые реализуют сложные сценарии поведения, например:

lets_talk(): Отправляет запрос в GeminiAssistant для общения.

weather_check(): Запрашивает погоду через GeminiAssistant и озвучивает ее.

wikisearch(): Ищет информацию в Wikipedia.

makeQR(): Генерирует QR-код.

get_face_frame(): Рисует рамку вокруг распознанных лиц.

calculate(): Простой калькулятор.

## 6. GeminiAssistant.py

ИИ для общения и получения информации.

ask(prompt): Отправляет текстовый запрос пользователя в Gemini с системной инструкцией  и возвращает ответ.

get_weather(): Использует бесплатный API Open-Meteo для получения прогноза погоды, парсит ответ и возвращает готовую фразу для озвучки.

## 7. Recognizer.py

Модуль для непрерывного распознавания речи в фоновом режиме с помощью Vosk.

## 8. Speaker.py

Модуль для озвучивания текста (Text-to-Speech).

Жестко настраивает переменные окружения для использования ALSA и звуковой карты (hw:3,0)

Инициализирует pygame.mixer для воспроизведения звука.

speak(text): Берет текст, генерирует MP3-файл через Google TTS (gTTS), проигрывает его через pygame.mixer и затем удаляет файл.

## 9. UART.py

Обеспечивает связь с микроконтроллером (Arduino, STM и т.д.) по UART для управления моторами, сервоприводами и чтения датчиков.

connect(): Сканирует порты, отправляет PING-команду и ждет ответа.

serial_send(data): Отправляет команду на микроконтроллер.

serial_receive(): Читает входящие данные.

getDistance(): Отправляет команду на ультразвуковой датчик и возвращает полученное расстояние.

## 10. ServoControl.py

Управление калибровочными углами сервоприводов.

Читает углы сервоприводов из файла angles.txt. Если файла нет, создает его со значениями по умолчанию (90).

getCalibrationCommand(): Формирует строку с калибровочными данными для отправки на микроконтроллер.

setAngle(data): Обновляет калибровочные углы в оперативной памяти (self.calibration_point) на основе команд, полученных от клиента.

## 11. Command.py

Хранилище всех команд, используемых для общения по UART и сокетам.


## 12. KeyPhraseAnalizer.py

Классификация голосовых команд для определения намерения пользователя.

Обучает простую ML-модель (SVM) на основе файла key_phrase.txt. В файле хранятся пары "текст_команды, ключевая_фраза".

analyzer(phrase): Принимает текстовую фразу, преобразует её в вектор с помощью TfidfVectorizer и предсказывает, к какой ключевой фразе (или фразам) она относится. Использует вероятности для определения порога срабатывания.

cut_stop_word(): Удаляет из фразы стоп-слова (предлоги, союзы и т.д.), загруженные из файла russian.

## 13. MusicPlayer.py / VideoPlayer.py / RadioPlayer.py

Отдельные виджеты для воспроизведения мультимедиа.

MusicPlayer использует QMediaPlayer для локальных файлов, ищет аудио в папке audio.

VideoPlayer использует QMediaPlayer и QVideoWidget для проигрывания видео. Имеет анимированное меню управления.

RadioPlayer использует VLC для потокового интернет-радио, так как QMediaPlayer с этим справляется плохо. Список станций читает из radio.txt.

Все три плеера имеют таймеры для автоматического скрытия элементов управления при бездействии и показываются/скрываются с помощью флагов *_active_trig.

## 14. FaceAction.py / Face.py

Модули для распознавания и идентификации лиц.

Используют OpenCV (LBPHFaceRecognizer) и каскады Хаара.

trainImage(): Проходит по папке dataset, находит лица на фотографиях и обучает (или дообучает) модель, сохраняя её в face.yml.

face_detect(): Детектирует лица на кадре, пытается их распознать с помощью обученной модели. Если уверенность низкая (<100), помечает как "unknown", иначе выводит имя пользователя из name.txt.

add_user(): Запускает диалог с пользователем для добавления нового лица. Спрашивает имя, затем делает 9 снимков, сохраняет их в папку и запускает переобучение модели.

## 15. HandAndFingerDetectionModule.py

findpostion(frame): Обрабатывает кадр, находит руки и возвращает список с координатами (x, y) для каждой ключевой точки (landmark) каждой руки.

findnameoflandmark(): Возвращает список названий обнаруженных точек (например, "THUMB_TIP", "INDEX_FINGER_DIP").

## 16. RockPapperScrissor.py

Реализация игры "Камень, ножницы, бумага" с использованием распознавания жестов.

Получает кадр с камеры (self.frame) из Server.

Использует HandAndFingerDetectionModule (через MediaPipe) для определения положения пальцев.

Интерпретирует комбинации пальцев как "Rock", "Paper" или "Scissors".

calculate_game_state(): Сравнивает жест игрока со случайным жестом компьютера и определяет победителя.

Взаимодействует с Recognizer для голосового диалога ("Сыграем еще раз?") и со Speaker для озвучивания результатов.

Управляет анимацией лица во время игры через флаги, которые считываются в main.py.

## 17. PlayVideoViaBrowser.py

Открытие веб-браузера с поисковым запросом на различных видеоплатформах.

find_video_browser(query, key_phrase): Анализирует ключевую фразу (youtube, vk, rutube) и очищает запрос от стоп-слов через KeyPhraseAnalizer.cut_stop_word().

Формирует URL для поиска на соответствующей платформе и открывает его в браузере по умолчанию с помощью webbrowser.open().

## 18. server_interface.py

Автоматически сгенерированный PyQt5 файл, описывающий статический пользовательский интерфейс главного окна.

Как работает: Содержит код, созданный из .ui файла. Определяет расположение кнопок, надписей и меток для видео (Video_label) и для наложения графики (outside_layout).
