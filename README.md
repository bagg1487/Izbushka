## О проекте

Полноценная прошивка для робота на базе микроконтроллера **STM32F4** с использованием **FreeRTOS**, драйверов **HAL** и **CMSIS**. 

Проект сгенерирован в **STM32CubeMX** для среды **Keil uVision** (MDK-ARM) и включает в себя:

- Управление 16 сервоприводами через две платы **PCA9685** (I2C)
- Управление двумя двигателями с ШИМ (L298N) с поддержкой **PID-регуляторов**
- Обработку команд по **UART** (DMA + прерывания)
- Измерение расстояния ультразвуковым дальномером **HC-SR04** (TIM3, Input Capture) x
- Чтение датчика температуры/влажности **DHT11/DHT22**  x
- Чтение **MPU6050** (акселерометр + гироскоп) с фильтром Калмана x
- RTOS-задачи: управление сервоприводами (ноги/руки отдельно), приём команд, управление моторами

**Возможности:**

- Дифференциальный привод с PID-регулятором скорости
- Одновременное и плавное движение всех сервоприводов
- Команды: движение, музыка, радио, "помахать", информация, меню
- Отправка телеметрии по UART (расстояние, датчики)
- Обработка кнопки 90° для входа в режим калибровки

## Периферия и пины

### I2C
| Шина | Адрес            | Устройство        | Пины STM32        |
|------|------------------|-------------------|-------------------|
| I2C1 | 0x68             | MPU6050           | PB6 (SCL), PB7 (SDA) |
| I2C2 | (не используется)| —                 | —                 |
| I2C3 | 0x40, 0x41       | PCA9685 (ноги/руки) | PA8 (SCL), PC9 (SDA) |

### PCA9685 (16 каналов, 48 Гц)
| Каналы | Плата       | Назначение        |
|--------|-------------|-------------------|
| 0–9    | PCA9685 #1 (0x40) | Руки (правая и левая) |
| 10–15  | PCA9685 #2 (0x41) | Ноги + поворот головы |

### Моторы (L298N)
| Мотор      | Направление 1   | Направление 2   | PWM (TIM1)      |
|------------|----------------|-----------------|-----------------|
| Motor1     | PG1, PE7       | PE7             | PE11 (CH2)      |
| Motor2     | PF13, PF14     | PF14            | PE13 (CH3)      |

### Ультразвук (HC-SR04)
| Сигнал     | Пин             | Таймер          |
|------------|-----------------|-----------------|
| TRIG       | PA7             | GPIO output     |
| ECHO       | PA6             | TIM3 CH1 (Input Capture) |

### Датчики
| Датчик          | Пин        | Периферия      |
|-----------------|------------|----------------|
| DHT11/DHT22     | PB5        | GPIO (pull-up) |
| Датчик освещения| PA1        | ADC1_IN1       |
| Reed switch     | PB4        | GPIO input     |

### Кнопки / переключатели
| Элемент         | Пин        |
|-----------------|------------|
| Кнопка 90°      | PB12       |

| Задача              | Стек    | Приоритет | Назначение                                   |
|---------------------|---------|-----------|----------------------------------------------|
| `defaultTask`       | 512     | normal    | Пустая (зарезервирована)                    |
| `ServoControlTas`   | 2048    | normal    | Управление ногами (ходьба, калибровка)       |
| `HandServoContro`   | 2048    | normal    | Управление руками (анимации)                 |

- `UARTDataMutex` — мьютекс для доступа к глобальному `state`

## Команды по UART (протокол)

Формат: `КОМАНДА#ПАРАМЕТР1#ПАРАМЕТР2\n`

| Команда                  | Параметры                | Описание                              |
|--------------------------|--------------------------|---------------------------------------|
| `CMD_SONIC`              | —                        | Отправить расстояние (HC-SR04)        |
| `CMD_SENSORS`            | —                        | Отправить все датчики (темп, влаж, reed, свет, расстояние, дым, тач) |
| `CMD_LIGHT`              | R,G,B (0–255)            | Управление RGB-светодиодом            |
| `CMD_PING`               | —                        | Ответ `CMD_PING`                      |
| `CMD_CALIBRATION`        | leg, servo, angle        | Установить угол калибровки            |
| `CMD_CALIBRATION_MOD`    | 1/0                      | Включить/выключить режим калибровки   |
| `CMD_CALIBRATION_ALL`    | 20 углов                 | Установить все углы (4×5)             |
| `CMD_MOVE_FORWARD`       | —                        | Вперёд                                |
| `CMD_MOVE_BACKWARD`      | —                        | Назад                                 |
| `CMD_MOVE_LEFT`          | —                        | Влево                                 |
| `CMD_MOVE_RIGHT`         | —                        | Вправо                                |
| `CMD_MOVE_STOP`          | —                        | Стоп                                  |
| `CMD_CLENCH_LEFT`        | —                        | Сжать левую руку (мотор)              |
| `CMD_CLENCH_RIGHT`       | —                        | Сжать правую руку                     |
| `CMD_LOOK_UP`            | —                        | Голова вверх                          |
| `CMD_LOOK_DOWN`          | —                        | Голова вниз                           |
| `CMD_LOOK_STOP`          | —                        | Стоп головы                           |
| `CMD_HOLD_3`             | mode (1–3)               | Idle-анимация через 3 минуты          |
| `CMD_HOLD_10`            | mode (1–3)               | Idle-анимация через 10 минут          |
| `CMD_HANDS_UP`           | —                        | Анимация «руки вверх»                 |
| `CMD_BOX`                | —                        | Анимация «бокс»                       |
| `CMD_DANCE`              | —                        | Анимация «танец»                      |

## Алгоритм обработки UART (кольцевой буфер + DMA)

1. Данные приходят по USART1 → DMA2 Stream2 (periph-to-memory)
2. Каждый байт вызывает колбэк `HAL_UART_RxCpltCallback`
3. Байты накапливаются в `tx_buffer` до `\r`
4. При `\r` ставится флаг `flag_send = true`
5. В задаче `UARTTask` буфер разбивается на строки (`strtok` с `\n`)
6. Каждая строка парсится: команда извлекается до `#`, параметры — циклом
7. Ответ (например, `CMD_SONIC#25\n`) отправляется через `HAL_UART_Transmit_DMA`

Функция `L298N_move_PID()` автоматически обновляет PID и устанавливает ШИМ.

## Анимации (конечные автоматы)

Глобальная переменная `state` управляет поведением:

| state | Режим                     |
|-------|---------------------------|
| 1     | Движение (вперёд/назад/поворот) |
| 2     | Idle-анимация 3 минуты    |
| 3     | Idle-анимация 10 минут    |
| 4     | Поворот на месте (hold3mod=1) |
| 5     | Руки вверх (приветствие)  |
| 6     | Ожидание / нейтральная поза |
| 7     | Наклон головы (вверх/вниз) |
| 8     | Режим калибровки          |
| 9     | Загрузка калибровочных углов из массива |
| 10    | (резерв)                  |
| 11    | Бокс                      |
| 12    | Танец                     |

## Быстрый старт

### 1. Установка инструментов

- Keil uVision 5 (MDK-ARM) — лицензия (Free для кода < 32KB)
- STM32CubeMX — генератор кода
- STM32Cube_FW_F4_V1.27.0 — пакет (установить через CubeMX)

### 2. Открытие и загрузка

```
# Скопировать папку проекта на компьютер
# Открыть STM32CubeMX → File → Load Project → izbushka_project_8_1.ioc
# Нажать "GENERATE CODE" 
# ИЛИ открыть Keil вручную: MDK-ARM/izbushka_project_8_1.uvprojx
```
